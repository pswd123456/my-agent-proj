# AGENTS.md

作用域：`docs/` 及其子目录。

## 文档目录规则

- `docs/` 优先承载长期有效的约定、决策、结构说明与工作流
- 新文档按主题拆分，避免长期把多个领域堆在同一文件
- 某个主题变大时，优先拆成“入口页 + 专题页”
- 文档应服务后续执行与协作，而不只是结论存档

## 主题分层

- `docs/architecture/` 放技术栈、架构边界、目录结构、工程决策
- `docs/design-system/` 放 UI 一致性、tokens、组件策略、页面模板、设计工作流
- `docs/template/` 放模板初始化、项目改名、目录裁剪与开发方式

## 维护约定

- 修改文档前，先检查当前目录及子目录是否有更具体的 `AGENTS.md`
- 规则若只属于某个子主题，应优先写入更具体的子目录
- 稳定长期使用的规则，应沉淀到对应文档而不是只留在对话里
- 更新文档时，尽量同步维护入口页与交叉链接
