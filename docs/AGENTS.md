# AGENTS.md

本文件作用域覆盖：

- `docs/`
- 以及其所有子目录

## 文档目录规则

- `docs/` 优先承载长期有效的约定、决策、结构说明与工作流
- 新文档应按主题拆分，避免把多个领域的规则长期堆在同一文件
- 当某个主题内容开始变大时，应优先拆成“入口页 + 专题页”，而不是继续追加到单篇长文
- 文档应优先服务后续执行与协作，不只做结论存档

## 当前主题划分

- `docs/architecture/` 放技术栈、架构边界、目录结构、工程决策
- `docs/design-system/` 放 UI 一致性、tokens、组件策略、页面模板、AI 设计工作流
- `docs/template/` 放模板初始化、项目改名、目录裁剪与单开发者工作方式

## 维护约定

- 修改文档前，先检查当前目录及子目录是否已有更具体的 `AGENTS.md`
- 若新增规则明显只属于某个子主题，应落到更具体的子目录文档中
- 若某条规则已经进入稳定长期使用状态，应优先写入对应文档，而不是只留在临时说明里
- 更新文档时应尽量同步维护索引页与交叉链接，避免入口失效
