# AGENTS.md

本文件作用域覆盖：

- `docs/design-system/`
- 以及其所有子目录

## 设计系统文档规则

- 设计系统相关内容应按主题拆分，不要重新堆回单篇总文档
- 优先按 `总览`、`tokens`、`组件与 patterns`、`页面模板`、`AI 工作流`、`回归与验收` 分开维护
- 新增规则时，先判断它属于哪个专题，再写入对应文档
- 如果某条约定同时影响实现和协作流程，应在专题文档中落规则，并在总览页增加链接

## 阅读顺序

1. 先读 `README.md`
2. 再读当前任务最相关的专题文档
3. 如果同时涉及 tokens、组件、模板或回归，按需补读相邻专题

## 与实现联动

- 设计系统文档应与 `packages/tokens`、`packages/ui`、`packages/ui-patterns` 的实现方向保持一致
- 若文档约定与代码现状冲突，应明确是修正文档还是修正实现，不要默认忽略冲突
