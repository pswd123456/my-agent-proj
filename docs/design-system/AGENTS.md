# AGENTS.md

作用域：`docs/design-system/` 及其子目录。

## 文档组织

- 设计系统内容按主题拆分维护，不要重新堆回单篇总文档
- 优先按总览、tokens、组件与 patterns、页面模板、工作流、回归验收分开整理
- 新增规则时，先判断归属专题，再写入对应文档
- 同时影响实现与协作流程的规则，在专题文档落地，并在总览页补链接

## 阅读顺序

1. 先读 `README.md`
2. 再读当前任务最相关的专题文档
3. 如同时涉及 tokens、组件、模板或回归，再补读相邻专题

## 与实现联动

- 文档应与 `packages/tokens`、`packages/ui`、`packages/ui-patterns` 的实现方向保持一致
- 若文档与代码冲突，先明确修正文档还是修正实现
