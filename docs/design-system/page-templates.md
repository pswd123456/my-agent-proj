# 页面模板

组件统一局部，模板统一页面。

## 使用规则

- 新页面优先映射到现有模板
- 若需求无法落入现有模板，先判断是否是模板变体，而不是全新模板
- 全新模板必须文档化，否则 AI 后续无法稳定复用

## 建议支持的模板

### `DashboardPage`

适用场景：

- 工作台总览
- 每日概览
- 关键指标首页

结构建议：

- 页头区
- 关键指标区
- 主要任务区
- 辅助信息区

### `ListDetailPage`

适用场景：

- 任务列表与详情
- 项目列表与详情
- 知识条目列表与详情

结构建议：

- 页头区
- 筛选与操作区
- 主列表区
- 详情侧栏或详情面板

### `EditorPage`

适用场景：

- 新建实体
- 编辑配置
- 调整流程参数

结构建议：

- 页头区
- 主要表单区
- 辅助说明区
- 底部操作区

### `ConversationPage`

适用场景：

- Copilot 对话
- agent 任务拆解
- 智能建议生成

结构建议：

- 顶部上下文区
- 消息流区
- 输入区
- 侧边上下文区

#### `ConversationPage / Workbench` 变体

适用场景：

- 左侧需要 `session` 切换
- 中间是主对话与流式执行
- 右侧需要并行展示日历、trace、prompt 或调试信息

结构建议：

- 顶部全局状态区
- 左侧 `session rail`
- 中间消息流与输入区
- 右侧只读上下文区，例如周历与调试 inspector

约束：

- 主消息流只保留用户、assistant 和精简后的执行活动
- 完整 prompt、thinking、tool result 原文放到 inspector，不直接塞满消息区
- 三栏布局在窄屏下应退化为纵向堆叠，而不是强行压缩

### `ReviewPage`

适用场景：

- 周期复盘
- 执行分析
- 结果总结与后续建议

结构建议：

- 页头区
- 总结区
- 时间线或复盘区
- 后续行动区
