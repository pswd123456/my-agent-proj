# AI 生成工作流

## 说明

根目录 `DESIGN.md` 已经负责高层视觉契约和禁止事项。

本页只描述 AI 在生成或改造页面时的执行顺序，避免重复解释同一套视觉原则。

## 生成前检查

生成页面前，AI 必须先检查：

1. 是否已有匹配的页面模板
2. 是否已有可复用组件
3. 是否已有对应 token 与语义状态
4. 是否只是现有页面的一个模板变体

## 落地约束

- 视觉方向、组件语气、禁止事项以 `DESIGN.md` 为准
- token 使用与新增判断以 `tokens.md` 为准
- 组件复用边界以 `components-and-patterns.md` 为准
- 页面布局优先使用 `page-templates.md` 中已有模板

## 标准输出顺序

要求 AI 先输出结构说明，再输出代码。

标准说明格式建议：

1. 该页面使用的模板名称
2. 复用的已有组件列表
3. 新增的组件或 variant
4. 是否新增 token
5. 是否影响已有页面模式

## Prompt 模板

```md
请基于现有设计系统实现这个页面，并严格遵守以下规则：

1. 先阅读仓库根目录 `DESIGN.md`，确认整体视觉契约与禁止事项。
2. 先列出将复用的现有组件，再开始写代码。
3. 先判断该页面应使用哪个既有 page template。
4. 仅在现有组件无法覆盖时新增 variant，不要轻易新建组件。
5. 页面中的颜色、圆角、阴影、间距必须通过项目 tokens 或设计系统变量实现。
6. 如果需要新增 token，先解释为什么无法复用现有 token。
7. 代码输出前先给出“模板选择 + 组件复用 + 新增项”的简短结构说明。
```
