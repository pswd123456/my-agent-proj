# AI 生成工作流

## 目标

- 让 AI 优先复用现有系统，而不是自由发挥页面
- 把页面生成过程收敛到可检查、可复用、可回归的流程里

## 生成前检查

生成页面前，AI 必须先检查：

1. 是否已有匹配的页面模板
2. 是否已有可复用组件
3. 是否已有对应 token 与语义状态
4. 是否只是现有页面的一个模板变体

## 硬约束

- 不直接写裸颜色值
- 不直接写随意的 `px` 间距与尺寸
- 不直接引入页面级一次性组件，除非证明有复用价值
- 不在业务页面重复实现已有交互模式
- 页面布局优先使用模板与 patterns

## 标准输出顺序

要求 AI 先输出结构说明，再输出代码。

标准说明格式建议：

1. 该页面使用的模板名称
2. 复用的已有组件列表
3. 新增的组件或 variant
4. 是否新增 token
5. 是否影响已有页面模式

## Prompt 模板

```md
请基于现有设计系统实现这个页面，并严格遵守以下规则：

1. 先判断该页面应使用哪个既有 page template。
2. 先列出将复用的现有组件，再开始写代码。
3. 仅在现有组件无法覆盖时新增 variant，不要轻易新建组件。
4. 不允许直接使用裸颜色、裸阴影、裸间距值，必须通过项目 tokens 或设计系统变量实现。
5. 页面结构必须与现有模板保持一致，不要重新发明布局。
6. 如果需要新增 token，先解释为什么无法复用现有 token。
7. 代码输出前先给出“模板选择 + 组件复用 + 新增项”的简短结构说明。
```
