# AGENTS.md

本文件作用域覆盖：

- `apps/`
- 以及其所有子目录

## 应用目录规则

- `apps/` 只放具体应用、运行入口或独立部署单元
- 仅当某段逻辑明显依赖某个应用运行时或交互壳层时，才应放在对应 app 内
- 若某段能力会被多个应用共享，应优先移动到 `packages/`
- 不要把跨端共享的领域模型、数据库 schema、agent 核心能力直接堆在某个应用内

## 当前目录建议

- `apps/web` 负责 `Web MVP`
- `apps/api` 负责对外 API 与应用层编排
- 当前设计默认只保留明确启用的应用目录；若未来确实需要独立后台进程，再新增对应 app

## 开工前检查

- 修改 `apps/` 内文件前，先回看根目录 `AGENTS.md`
- 做 UI 相关工作前，先阅读 `docs/design-system/` 下对应文档
- 做架构、接口边界、工程结构相关工作前，先阅读 `docs/architecture/` 下对应文档
- 做模板初始化或裁剪应用目录时，先阅读 `docs/template/` 下对应文档
