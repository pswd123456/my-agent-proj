# AGENTS.md

作用域：`apps/` 及其子目录。

## 目录职责

- `apps/` 只放具体应用、运行入口或独立部署单元
- 明显依赖某个应用运行时或交互壳层的逻辑，才放在对应 app 内
- 可被多个应用复用的能力，应优先下沉到 `packages/`
- 不要把跨端共享的领域模型、数据库 schema、agent 核心能力堆在某个 app 内

## 开工前检查

- 先回看根目录 `AGENTS.md`
- 做 UI 相关工作前，先看 `DESIGN.md` 与 `docs/design-system/`
- 做架构、接口边界、工程结构相关工作前，先看相关架构文档
- 做模板初始化或应用目录裁剪前，先看 `docs/template/`
