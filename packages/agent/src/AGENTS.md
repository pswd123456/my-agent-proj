# AGENTS.md

本文件作用域覆盖：

- `packages/agent/src/`
- 以及其所有子目录

## 源码层规则

- `src/` 只放 `@ai-app-template/agent` 的源码实现、内部适配层和对外入口
- 顶层文件优先保留为入口或 barrel，具体实现优先下沉到子目录
- 一个文件优先只承载一个清晰职责，避免把契约、实现、工具函数和导出入口混写在一起
- 共享逻辑优先放在就近的 `shared`、`contracts` 或 `index` 文件中，减少跨层跳转
- `session/` 目录只放会话契约、共享校验、文件实现、内存实现和相关 barrel
- `tools/` 目录只放工具契约、注册表、通用工作区辅助和具体工具实现
- 如果某个模块开始继续变大，优先先拆到更小文件，再考虑引入新的子目录
- 新增长期约定时，应优先更新本文件，避免规则只留在临时对话里

## 架构说明

- `runtime.ts` 负责 agent 执行编排，不负责具体 session 持久化或工具实现
- `prompt.ts` 负责 prompt 拼装和消息转换，不负责业务状态持久化
- `session/` 负责 session 的抽象、校验和存取实现
- `tools/` 负责 tool registry 和每个具体 tool 的实现
- `model.ts` 负责模型协议适配，后续若接入更多 provider，再考虑进一步拆 provider 目录

## 可更新约定

- 这些规则是活文档，可以按架构演进、职责迁移和新增实现方式继续更新
- 当目录边界变化时，优先先改这个文件，再改代码中的导出和 import
